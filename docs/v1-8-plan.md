# v1.8 Plan — 新手情境入口與開發進度看板

版本：v1.8 規劃草案
狀態：待使用者決定是否啟動實作

## 目標

v1.8 聚焦兩個新手入口：

1. 「我要做什麼」：讓新手不用理解提示詞庫分類，直接依情境選提示詞。
2. 「開發進度」：讓使用者打開控制台就知道專案目前做到哪裡、下一步是什麼、是否卡住。

本文件只規劃 MVP，不實作程式碼。

## MVP A：「我要做什麼」Tab

### 使用者痛點

新手看到「提示詞庫」容易不知道該選哪張卡。情境入口應該用白話問題引導，而不是要求使用者理解分類、flow 或 stage。

### 資料設計

新增檔案：

```text
data/scenarios.yaml
```

第一版欄位：

```yaml
- id: build-app
  title: 從零做一個完整應用
  plain_zh: 我想從零開始做一個可以使用的完整系統。
  prompt_titles:
    - 從零開始的完整應用
```

欄位說明：

| 欄位 | 說明 |
|---|---|
| `id` | 穩定識別用，不顯示給使用者 |
| `title` | 情境卡標題，用白話描述使用者想做什麼 |
| `plain_zh` | 小白版說明，幫使用者確認這張是不是他要的 |
| `prompt_titles` | 對應既有提示詞卡標題，可一對多 |

### 第一版情境

先做 8–10 個情境：

1. 從零做一個完整應用
2. 開發一個生產級功能
3. 修 bug / 找根本原因
4. 理解並重構舊專案
5. 優化速度與效能
6. 設計系統架構並實作
7. 建立可重用 UI 元件
8. 開發可發布 API
9. 多 Agent / 多 AI 分工
10. 清晰重建專案架構

### 互動設計

- 新增 tab：「我要做什麼」。
- 每張情境卡顯示：
  - 情境標題
  - 小白版說明
  - 對應提示詞數量
  - 「查看提示詞」按鈕
- 點擊情境後：
  - 切到「提示詞庫」
  - 清空搜尋
  - 高亮對應 prompt cards
  - 優先重用 v1.7 的 `data-prompt-key` 機制

### 非目標

- 不改 `data/prompts.yaml` 既有 schema。
- 不在 v1.8 第一版自動產生提示詞。
- 若需要新增提示詞，走「收錄新內容」流程。

## MVP B：「開發進度」Tab

### 使用者痛點

使用者開發系統時會想知道：

- 現在做到哪個階段？
- 上一步完成了什麼？
- 下一步是誰要做？
- AI 是不是卡住了？
- 要複製什麼提示詞繼續推進？

目前二刀流中控已有 `DUAL-AI-STATE` 貼上看板，但入口太深，而且需要手動貼文字。v1.8 應該讓本 repo 的狀態更直觀。

### 建置策略

`scripts/build.py` 在 build 時讀取：

```text
DUAL-AI-STATE.md
NEXT-AI-TASK.md
```

並 inline 成兩個前端常數：

```javascript
const STATE_HTML = "...";
const NEXT_HTML = "...";
```

可沿用既有 markdown card 轉換邏輯，或新增輕量 `markdown_to_cards` 包裝。

### 摘要卡

「開發進度」tab 第一版顯示：

| 區塊 | 來源 |
|---|---|
| 目前階段 | `DUAL-AI-STATE.md` 的「目前階段」 |
| 已完成 | `DUAL-AI-STATE.md` 的「已完成事項」摘要 |
| 下一步 | `DUAL-AI-STATE.md` 的「下一步」 |
| 未解決問題 | `DUAL-AI-STATE.md` 的「未解決問題」 |
| 最後更新 | `DUAL-AI-STATE.md` 的「最後更新時間」 |
| 下一棒 AI | `NEXT-AI-TASK.md` 的「下一棒 AI」 |

### 手動 Fallback

保留現有二刀流中控的貼上 textarea 路徑，作為：

- 查看其他專案狀態
- 跨裝置接續
- 狀態檔尚未被 build inline 時的臨時替代

### 非目標

- 不直接讀任意資料夾。
- 不在瀏覽器中修改 `DUAL-AI-STATE.md`。
- 不做背景常駐監控。
- 不自動判斷任務是否卡住；第一版只顯示狀態檔已寫出的卡點。

## 建議實作順序

1. 先做「開發進度」tab，因為它直接解決使用者焦慮，也能幫後續開發更容易接續。
2. 再做 `data/scenarios.yaml` 與「我要做什麼」tab。
3. 最後再把 10 個 Claude 工程提示詞整理成可收錄卡片；若要新增提示詞，走「收錄新內容」流程。

## 驗收標準

- `python3 scripts/build.py` 成功。
- 打開 `index.html` 可看到「開發進度」tab。
- 「開發進度」tab 能顯示目前階段、下一步、未解決問題、最後更新、下一棒 AI。
- 打開「我要做什麼」tab 可看到 8–10 個情境卡。
- 點情境卡可切到提示詞庫並高亮對應提示詞卡。
- 不修改 `data/prompts.yaml` 既有 schema。

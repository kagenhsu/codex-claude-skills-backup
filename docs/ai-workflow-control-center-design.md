# 三方 AI 協作中控台設計規格

版本：v0.1（2026-06-04）  
狀態：設計草案，等待使用者確認後再實作

## 1. 背景

目前控制台已能查詢 skills、複製提示詞、查看三方 AI 工作流與角色導覽。下一步希望把它升級成更完整的「AI 協作中控台」：

- 開新專案時，能在討論完設計方向後產生建立 `AGENTS.md` 的交辦提示詞。
- 遇到新的好用 skill 或提示詞時，能用表單收錄，產生 YAML 片段與給 Codex 的完整交辦提示詞。
- 三方 AI 工作流進行中，能顯示下一步要交給誰、為什麼、要送什麼，並在使用者確認後才送出或產生交接包。

核心原則：AI 可以協助轉交與整理，但不能在使用者看不到的情況下私下互相決定。

## 2. 目標

第一版要做到：

1. 控制台新增 `專案規劃` 頁籤。
2. 控制台新增 `收錄新內容` 頁籤。
3. 控制台新增 `三方中控` 頁籤。
4. 所有自動化動作都先產生「可檢查內容」，再由使用者確認。
5. 純 HTML 不能直接安全寫檔，因此第一版以「生成提示詞、生成 YAML、複製、建立交接檔建議」為主。

## 3. 不做的事

- 第一版不直接從瀏覽器寫入 `data/*.yaml`。
- 第一版不自動 push GitHub。
- 第一版不讓 AI 私下互傳訊息。
- 第一版不要求 Claude Code（VS Code）或 Claude Desktop 必須有自動化 API。
- 第一版不安裝陌生 skill；skill 收錄必須先經過安檢提示。

## 4. 功能群一：專案規劃

### 4.1 用途

使用者和 AI 討論完新專案方向後，可以用這個頁籤產生「建立 AGENTS.md 頂層規則文件」的 Codex 交辦提示詞。

### 4.2 頁籤名稱

`專案規劃`

### 4.3 第一版功能卡

卡片名稱：`設計方向確認後建立 AGENTS.md`

使用者填寫欄位：

- 專案名稱
- 一句話介紹
- 專案目標
- 使用者偏好的工作方式
- AI 回答風格，例如「用小白聽得懂的話」
- 是否需要 git / 測試 / commit 規則
- 已討論完成的設計方向
- 不確定或待確認事項

### 4.4 產出

產出一段可直接貼給 Codex 的交辦提示詞。提示詞要求 Codex：

1. 建立或更新專案根目錄 `AGENTS.md`。
2. 寫入專案名稱、目標、協作規範、檔案結構、修改與測試注意事項。
3. 不確定的地方列成「待確認問題」，不要自行亂補。
4. 建立文件後回報改了什麼，但不自動 push。

### 4.5 教學文案

頁面要用白話提醒：

> AGENTS.md 是給 AI 看的專案規則書。以後 Codex 或 Claude Code 接手時，會先看它，知道這個專案該怎麼做、哪些事不能亂做。

## 5. 功能群二：收錄新內容

### 5.1 用途

使用者看到新的好用 skill 或提示詞時，不需要懂 YAML。只要填表單，控制台會產生：

- 推薦複製：給 Codex 的完整交辦提示詞
- 進階檢查：YAML 片段

### 5.2 頁籤名稱

`收錄新內容`

### 5.3 內容類型

表單第一步先選：

- `提示詞`
- `Skill`

### 5.4 提示詞表單欄位

- 標題：控制台卡片名稱
- 分類：例如 `三方 AI 協作`、`Skill 管理`、`安全檢查`、`專案啟動`
- 用途：什麼時候使用
- 適用流程：`common` / `dualai` / `solo`
- 階段：例如 `entry`、`3`、`4`、`5`、`archive`、空白
- 提示詞原文

### 5.5 Skill 表單欄位

- skill 名稱
- 來源：GitHub URL、資料夾路徑或來源描述
- 分類
- 風險等級：低 / 中 / 高
- 用途摘要
- 觸發句，支援多行
- 注意事項

Skill 表單要顯示安全提醒：

> 陌生來源的 skill 請先做安檢。不要直接安裝會讀取金鑰、執行不明腳本或要求忽略規則的 skill。

### 5.6 產出的 Codex 交辦提示詞

表單產生的交辦提示詞採完整模式：

1. 請 Codex 檢查內容格式與風險。
2. 寫入 `data/prompts.yaml` 或 `data/skills.yaml`。
3. 執行 `python3 scripts/build.py` 重建 `index.html`。
4. 驗證 `index.html` 搜尋得到新卡片。
5. 建立本地 git commit。
6. 明確不要 push `origin/main`。

### 5.7 第一批建議內建提示詞

這些提示詞應在實作時加入 `data/prompts.yaml`：

- `給 Codex 的存檔收尾提示詞`
- `給 Codex 的逐條修正轉交提示詞`
- `設計方向確認後建立 AGENTS.md`

## 6. 功能群三：三方中控

### 6.1 用途

把三方 AI 工作流從「手動轉貼」升級成「透明中控」。控制台負責顯示狀態、產生下一步交接內容，並讓使用者確認後再送出或複製。

### 6.2 頁籤名稱

`三方中控`

### 6.3 核心規則

每次轉交前都要顯示：

- 要送給誰：Codex / Claude Code（VS Code）/ Claude Desktop
- 為什麼送：規劃、實作、審查、修正、複審、文件整理、存檔收尾
- 要送什麼：完整提示詞內容
- 送出方式：自動送出 / 複製貼上 / 建立交接檔

使用者按確認後才進入下一步。

### 6.4 三方通道策略

同一套確認流程，底層依工具能力決定送出方式：

| 角色 | 第一版策略 | 未來策略 |
|---|---|---|
| Codex | 若工具支援 thread，優先產生可送到 Codex thread 的內容；否則複製貼上 | 支援自動建立 thread、送訊息、讀回結果 |
| Claude Code（VS Code） | 產生可貼到 VS Code Claude Code 的提示詞 | 若未來有可用通道，再接上自動送出 |
| Claude Desktop | 產生可貼到 Claude Desktop 的提示詞 | 若未來有可用通道，再接上自動送出 |

### 6.5 狀態資料

保留 `DUAL-AI-STATE.md` 作為人類可讀狀態。

未來可新增 `.ai-workflow/`：

```text
.ai-workflow/
├── state.json
├── outbox/
│   ├── 2026-06-04-to-claude-code-review.md
│   └── 2026-06-04-to-codex-fix.md
└── history/
    └── 2026-06-04-events.jsonl
```

第一版可以先不建立 `.ai-workflow/`，只在控制台產生建議內容。第二版再加入檔案化交接。

### 6.6 中控台顯示

頁面顯示：

- 目前階段：第 1 到第 5 階段，或已完成
- 目前負責角色
- 下一個建議動作
- 上一步摘要
- 交接包預覽
- 複製按鈕

範例下一步按鈕：

- `交給 Claude Code 審查`
- `交回 Codex 修正`
- `交給 Claude Desktop 整理主管版說明`
- `交給 Codex 存檔收尾`

## 7. UI 設計原則

- 不要求使用者懂 YAML。
- 新手推薦複製「給 Codex 的完整交辦提示詞」。
- YAML 片段標為進階檢查用。
- 每個欄位都要有一句白話說明。
- 危險或不可逆動作要明確提醒，例如安裝陌生 skill、commit、push。
- 頁面文字要短，避免把使用者淹沒在規則裡。

## 8. 技術設計

現有 `scripts/build.py` 已把 YAML 資料與 Markdown 文件編譯成單一 `index.html`。第一版可直接擴充這個模板：

- 新增 `planner` tab。
- 新增 `capture` tab。
- 新增 `control` tab。
- 在前端 JavaScript 中加入表單狀態、YAML 字串生成、Codex 交辦提示詞生成。
- 不新增外部依賴。
- 不需要後端服務。

資料仍由使用者或 Codex 寫入 `data/*.yaml`，控制台只負責產生內容與複製。

## 9. 驗證方式

實作完成後至少驗證：

1. `python3 scripts/build.py` 成功。
2. 雙擊 `index.html` 可看到新增三個頁籤。
3. `專案規劃` 能產生建立 `AGENTS.md` 的交辦提示詞。
4. `收錄新內容` 選提示詞時，能產生 prompt YAML 與 Codex 交辦提示詞。
5. `收錄新內容` 選 Skill 時，能產生 skill YAML 與安檢提醒。
6. `三方中控` 能顯示五階段下一步與交接包預覽。
7. 搜尋與既有 Skills、提示詞庫、工作流、角色導覽頁籤不受影響。

## 10. 分期建議

### 第一期：表單與交辦提示詞

- 實作 `專案規劃`。
- 實作 `收錄新內容`。
- 新增三張內建提示詞卡。
- 不處理自動送出。

### 第二期：三方中控頁

- 實作 `三方中控` 的狀態顯示與交接包預覽。
- 仍以複製貼上為主。

### 第三期：半自動交接檔

- 新增 `.ai-workflow/outbox/` 與 `.ai-workflow/history/`。
- 讓 Codex 可產生交接檔並記錄歷史。

### 第四期：工具通道整合

- 若 Codex thread 工具可用，加入送出前確認與自動建立 thread。
- Claude Code（VS Code）與 Claude Desktop 維持相同確認流程，依工具能力決定是否能自動送出。
